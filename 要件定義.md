## 1. 概要

エホバの証人・徳山会衆（周南市）の**週日の集会（クリスチャン生活と奉仕の集会）**における、各プログラムの担当割り当てを支援するデスクトップアプリを開発する。

会衆内の**責任者1名**が、各集会日のプログラムを「そのプログラムを果たせる資格（ロール）を持つ成員」に割り当てる作業を支援する。過去の担当履歴をもとに「前回担当してから最も間が空いている人」を優先表示することで、公平かつ効率的な割り当てを実現する。

### 目的・ゴール
- 成員ごとの担当可能プログラム（ロール）を一元管理できる。
- 割り当てを、履歴に基づく優先度つきプルダウンで簡単に行える。
- 集会プログラムは Excel から取り込み、割り当て済み Excel として書き出せる。
- 過去の記入済み実績（PDF）から履歴を初期投入でき、導入直後から優先度が機能する。

### 利用形態（確定）
- **操作者は非技術者の責任者1名**。**PC**で利用。単一利用（同時編集・ログイン認証・サーバなし）。
- 最終成果物は**名前入り Excel**。会衆への共有・PDF化は既存手段に委ねる。

## 2. 用語定義（グロッサリ）

| 用語 | 定義 |
|------|------|
| 成員（メンバー） | 会衆に所属する人物。割り当ての対象者。 |
| プログラム | 集会内の個々の項目（例: 聖書朗読、会衆聖書研究）。表示上の1項目。 |
| **スロット** | **割り当ての最小単位**。1プログラムは1つ以上のスロットを持つ（例: 会衆聖書研究 =「司会」スロット＋「朗読」スロット）。**1スロット = 1ロール = 1成員 = 1履歴**。 |
| ロール | あるスロットを果たす資格。候補プールを決める。既定で**履歴もロール単位**で蓄積。 |
| ロールグループ | 成員へロールを**一括付与**するための束（事務的便宜）。優先度計算とは無関係。 |
| **優先度グループ**（v1「プログラムグループ」） | 複数ロールの履歴を**合算**して優先度計算するための束。 |
| ペア履歴 | 実演スロットの「演者 × 相手役」の組み合わせと担当日の記録。相手役の並び順に用いる。 |
| 割り当て | ある集会日の、あるスロットを、ある成員に担当させること。 |
| 責任者 | 割り当てを行う操作者（1名）。 |
| 会場 | スロットの実施場所（本会場／補助教室）。v1は本会場のみ。将来拡張の予約属性。 |

## 3. 技術・配布・保存（確定）

- **フレームワーク: Tauri v2 デスクトップアプリ**（Windows / Mac 両対応、.exe/.app をダブルクリック起動）。
  - フロントは HTML/TypeScript。業務ロジックはフロント側。Rust 部分は薄いシェル。
  - Mac 版バイナリは macOS 実機または CI（GitHub Actions macOS ランナー）でビルド（Windows からのクロスコンパイル不可）。未署名のため初回起動時の SmartScreen / Gatekeeper 回避手順を操作者へ案内。
- **保存: IndexedDB ＋ `data.json` の二重書き込み（自動）**。
  - IndexedDB が破損した場合は `data.json` から復旧。
  - PC 移行は**アプリのフォルダ（実行ファイル＋ data.json）をコピー**するだけ。
  - 加えて手動バックアップ（任意タイミングでの JSON 書き出し）を提供。
- **Excel 読み書き: ExcelJS**（既存の書式・結合セルを保持して書き戻す）。

## 4. 機能要件

### 4.1 成員マスター管理
- 成員の登録・更新。フィールド: **氏名（フルネーム）／性別（男・女）／状態（在籍・非活動/転出）／保持ロール[]／保持ロールグループ[]**。
- ロールは**ロールグループによる一括付与**が可能（「こういうタイプの人」に相当）。
- **転出入・資格変更は「現在状態の編集」で表現**。成員は削除せず状態変更で候補から外す。**過去履歴は不変**（遡及修正は初版対象外）。

### 4.2 ロール／ロールグループ／優先度グループ管理
- **ロール**を定義（スロットに対応。候補プールと履歴単位を規定）。
- **ロールグループ**を定義（複数ロールを内包。成員へ一括付与）。
- **優先度グループ**を定義（複数ロールの履歴を合算）。すべてユーザーが編集可。
  - 初期の優先度グループ（変更可）: `宝 part1 + part2`／`生活の討議・話（part7 系）`。
  - 野外奉仕 part4/5/6 は**同一の生徒ロール**とするため履歴共有は自動（優先度グループ不要）。

### 4.3 集会プログラムの取り込み（Excel）
- **1シート = 2週分**の構成。取り込み時に日付行で分割し、内部は**1集会日単位**で扱う。
- **方式 = 自動検出 ＋ レビュー確定**。
  - E列ラベルは不完全（part1/2/7、多くの part8 で欠落）のため、判定は **C列キーワード（"聖書朗読"/"会衆の聖書研究"/"祈り" 等）＋セクション見出し＋ part 番号＋ E列ラベル**を併用。
  - 検出した各行に**型と既定ロール・スロット数を仮割り当て**し、レビュー画面で責任者が確認・修正。修正は**型シグネチャ単位で記憶**し次回自動適用。
  - 未認識行は「未分類」として明示し、無視または手動分類。
  - 巡回訪問週（会衆聖書研究→奉仕の話＝巡回監督担当で**非割当**）、特別キャンペーン等の臨時項目に対応。part6 が「話」の回は**相手役スロットを省略可**。

### 4.4 割り当て画面
- **1画面 = 1集会日**。各スロットをプルダウンで割り当て。
- 候補 = そのスロットのロール保持者。並び順は優先度順（§4.6）。**あくまで提案**であり、候補プール内の誰でも選択可。
- **同一日の重複割当**は確認ダイアログを1回表示（ハード除外はしない）。
- **保存**でアプリDBへ確定（履歴に反映）。優先度への反映は保存時（入力途中で即時再ソートしない）。
- **保存後の修正**: 保存済み日付を再オープン→変更→再保存。古い割当を履歴から取り消して上書きし整合を保つ。

### 4.5 エクスポート（Excel 出力）
- **保存とは別操作**。1か月分がそろった段階等で共有用ファイルを生成。
- 各日付の割当を**元ワークブック（1シート2週分）の該当 E列へ書き戻し**、**別名で新規保存**（原本テンプレは保持）。
- 書き戻し形式: E列ラベルの後ろに**名前を追記（敬称なし）**、2枠（生徒/相手）は **`/` 区切り**。
- v1 は **Excel 出力のみ**。PDF 化は従来手段（Excel から印刷/PDF保存）に委ねる。

### 4.6 優先度計算
- 各スロット候補を**「そのロール（または優先度グループ）の最終担当日が古い順」**に提示。
- **未担当者 = 最優先（最上位）**。
- **タイブレーク**: ①通算担当回数が少ない順 → ②ランダム（固定化防止）。
- **相手役スロット**の並び: **主キー = その演者とのペア最終担当日（未ペア = 最上位）／副キー = 相手役個人の最終担当日**。→ 同ペア固定化と相手役集中の両方を回避。
- **性別**: 兄弟限定ロールはロール付与で表現。相手役は**既定で同性に絞り**、「全員表示」で夫婦など異性も選択可（ハード強制なし）。
- **野外奉仕の「話」**: part4-6 の生徒枠には**「兄弟のみ」チェックボックス**を備え、候補を男性に絞れる。part6「話」の回（相手役省略）を指定すると**自動でオン**（手動で解除可・ハード強制なし）。ロールは生徒ロール1本のまま（履歴共有は §4.2 のとおり自動）。

### 4.7 履歴の初期投入（コールドスタート）
- **直近3か月**（例: 2026年4〜6月）の**記入済み PDF**（テキスト抽出可・Excel と同一レイアウト）から履歴をインポート。
- 名前は**フルネームでほぼ全自動名寄せ**。曖昧時のみ責任者が対象成員を選択（一度対応づけたら記憶）。
- 取り込み・出力・履歴インポートいずれも**1シート2週分**構造で扱う。

## 5. データモデル（案）

- **Member（成員）**: id, 氏名, 性別, 状態, 保持ロール[], 保持ロールグループ[]
- **Role（ロール）**: id, 名称, 対象スロット種別, （所属優先度グループ 任意）
- **RoleGroup（ロールグループ）**: id, 名称, 含まれるロール[]
- **PriorityGroup（優先度グループ）**: id, 名称, 含まれるロール[]
- **Program / Slot（プログラム・スロット）**: プログラム(id, 名称, 会場) と スロット(id, 所属プログラム, 要求ロール, 種別〈演者/相手役 等〉)
- **Meeting（集会日）**: id, 開催日, プログラム構成（Excel 由来）
- **Assignment（割り当て）**: id, 集会日, スロット, 割り当て成員, 担当日
- **PairAssignment（ペア履歴）**: 実演の 演者Assignment × 相手役Assignment, 担当日

### 優先度計算のための派生データ
- 成員 × ロール（または優先度グループ）ごとの**最終担当日**（グループは含まれるロールいずれかの最新日）。
- 成員 × ロールごとの**通算担当回数**（タイブレーク用）。
- 演者 × 相手役ごとの**最終ペア担当日**（相手役の並び用）。

## 6. ビジネスルール

1. プルダウン候補 = 対象スロットのロールを持つ**在籍**成員のみ。
2. 並び順 = 最終担当日が最も古い成員を上位（未担当者最優先）。同着は 通算回数少ない順 → ランダム。
3. 最終担当日の判定単位: ロールが優先度グループに属する場合はグループ単位、属さない場合はロール単体。
4. 相手役の並び = 演者とのペア最終担当日（主）→ 相手役個人の最終担当日（副）。
5. 相手役の候補 = 既定は演者と同性、「全員表示」で異性も可。
6. ロールグループ付与 = 含まれる全ロールが有効化。
7. 同一集会日の重複割当 = 確認ダイアログを提示（禁止はしない）。
8. 保存 = 履歴へ確定。保存後の再編集で履歴を整合的に上書き。
9. 転出入・資格変更 = 現在状態の編集で表現。履歴は不変。
10. 野外奉仕（part4-6）の生徒枠 = 「兄弟のみ」チェックで候補を男性に絞り込める。part6「話」の回は自動でオン（手動変更可・禁止はしない）。

## 7. 画面要件

### 7.0 画面一覧（サマリ）

| # | 画面 | 主な機能 |
|---|------|----------|
| S1 | ホーム（集会日一覧） | 取り込み済み集会日の一覧と割り当て状況の俯瞰、各画面への入口 |
| S2 | 成員マスター管理 | 成員の登録・更新、性別・状態、ロール／ロールグループ付与 |
| S3 | ロール／ロールグループ設定 | ロール定義、ロールグループ定義と内包ロール設定 |
| S4 | 優先度グループ設定 | ロールの履歴合算グループの定義 |
| S5 | Excel 取り込み（自動検出＋レビュー） | 集会プログラムのインポートと型/ロール確認・修正 |
| S6 | 履歴インポート | 過去 PDF からの実績取り込みと名寄せ |
| S7 | 割り当て | 集会日ごとの各スロットへの割り当て（優先度つきプルダウン）、保存 |
| S8 | エクスポート | 割当済みワークブックの書き出し（別名保存） |
| S9 | バックアップ・設定 | JSON 手動バックアップの書き出し／復元 |

共通事項:
- 全画面に共通のナビゲーション（サイドメニュー等）から S1〜S9 へ遷移できる。
- 操作者は非技術者1名。専門用語を避けたラベル・確認ダイアログ・元に戻せる操作を基本とする。
- 保存を要する画面で未保存のまま離脱しようとした場合は確認ダイアログを表示する。

### S1. ホーム（集会日一覧）

起動直後に表示する画面。日常運用の起点。

**表示**
- 取り込み済みの集会日を日付順に一覧表示。各行: 開催日／ステータス（未割当・一部割当・割当済み・エクスポート済み）／割当済みスロット数「n / 全スロット数」。
- 巡回訪問週など特殊な週はその旨をバッジ表示。
- データが空（初回起動）の場合は、セットアップ手順（§8）への誘導を表示。

**操作**
- 集会日をクリック → S7 割り当て画面（その日）を開く。
- 「Excel を取り込む」ボタン → S5 へ。
- 「エクスポート」ボタン → S8 へ。

### S2. 成員マスター管理

**表示**
- 成員の一覧表: 氏名（フルネーム）／性別／状態（在籍・非活動/転出）／保持ロール（ロールグループ由来含む）。
- フィルタ: 状態（既定は在籍のみ表示）、ロール、氏名部分一致検索。

**操作**
- 新規登録: 氏名・性別・状態を入力し、ロールを個別付与またはロールグループで一括付与。
- 編集: 上記フィールドの変更。状態を「非活動/転出」へ変更すると以後の候補プルダウンから外れる（過去履歴は不変・削除機能なし → §4.1）。
- ロール付与の操作: 個別ロールのチェックボックス＋ロールグループの選択（グループ選択時は内包ロールが一括で有効化されることを画面上で明示）。

### S3. ロール／ロールグループ設定

**表示**
- ロール一覧: 名称／対象スロット種別／所属優先度グループ（あれば）／保持者数。
- ロールグループ一覧: 名称／内包ロール。

**操作**
- ロールの追加・名称変更・削除（保持者や履歴が存在する場合は削除不可または警告）。
- ロールグループの追加・編集: 内包ロールの選択（複数チェック）。
- 初期データとしてサンプル Excel の構造（§11）に対応する既定ロール一式を投入済みとし、この画面で変更可能とする。

### S4. 優先度グループ設定

**表示**
- 優先度グループ一覧: 名称／含まれるロール。
- 初期値（変更可）: 「宝 part1 + part2」「生活の討議・話（part7 系）」。

**操作**
- グループの追加・編集・削除: 含めるロールを複数選択。
- 1ロールが複数の優先度グループに属することは禁止し、既に所属済みのロールを選んだ場合は警告する。

### S5. Excel 取り込み（自動検出＋レビュー）

2段階（①ファイル選択 → ②レビュー確定）のウィザード形式。

**表示（①ファイル選択）**
- ファイル選択ダイアログ（.xlsx）。選択後、含まれるシートと分割された集会日（1シート=2週分 → 日付単位）を一覧表示。
- 既に取り込み済みの日付が含まれる場合は上書き確認を表示。

**表示（②レビュー）**
- 集会日ごとに、検出した各行を表形式で表示: 元セル内容（C列等）／判定した型／既定ロール／スロット数（1枠・2枠）／会場。
- 未認識行は「未分類」として目立つ形で明示（§4.3）。
- 巡回訪問週の非割当項目（奉仕の話）、臨時項目は「割当対象外」として表示。

**操作**
- 各行の型・ロール・スロット数を修正（プルダウン）。修正は型シグネチャ単位で記憶され次回から自動適用。
- 未分類行: 「無視する」または手動で型を割り当てる。
- part6 が「話」の回: 相手役スロットの省略を行単位で指定できる。
- 「確定して取り込む」 → 内部DBへ集会日・プログラム・スロットを登録し、S1 へ戻る。

### S6. 履歴インポート

**表示**
- PDF ファイル選択（複数可・1シート2週分構造）。
- 抽出結果のプレビュー: 集会日／スロット／抽出した氏名／名寄せ結果（自動確定 or 要確認）。
- 名寄せが曖昧な氏名は「要確認」として一覧の先頭に集約表示。

**操作**
- 要確認の氏名ごとに、成員プルダウンから対象者を選択（「新規成員として登録」も選択可）。一度対応づけた表記は記憶され再質問しない（§4.7）。
- 「確定」 → 履歴（Assignment／PairAssignment）として登録。
- 取り込み済みと重複する集会日の履歴は上書き確認。

### S7. 割り当て（1画面 = 1集会日）

本ツールの中核画面。

**表示**
- ヘッダ: 開催日／前日・翌日への移動／保存状態（未保存変更ありの明示）。
- 集会プログラムを Excel と同じ順序で縦に一覧表示。各行: プログラム名／スロット（1枠 or 演者+相手役の2枠）／割り当てプルダウン。
- プルダウン候補: そのスロットのロールを持つ在籍成員のみを優先度順（§4.6）に表示。各候補に**最終担当日**（未担当は「未」）を添えて表示し、並び順の根拠が見えるようにする。
- 相手役プルダウン: 演者選択後に有効化。並びはペア最終担当日（主）→個人最終担当日（副）。既定は演者と同性のみ表示。
- 巡回訪問週の非割当スロットは編集不可のグレー表示。

**操作**
- 各スロットのプルダウンで成員を選択。選択は提案順に縛られず候補プール内の誰でも可。
- 相手役の「全員表示」トグルで異性（夫婦等）も候補に含める。
- 野外奉仕（part4-6）の生徒プルダウン下の「兄弟のみ」チェックで候補を男性に絞る。「この回は話」を指定すると自動でオン（手動で変更可）。絞り込みにより候補から外れた選択済み成員は「（絞り込み対象外）」として保持・明示する。
- 同一日に既に割り当て済みの成員を選択した場合、確認ダイアログを1回表示（続行可 → §6-7）。
- 「保存」: 全スロットの割当を履歴へ確定。入力途中の即時再ソートは行わず、優先度への反映は保存時（§4.4）。
- 保存済みの日を再オープンして変更 → 再保存で履歴を整合的に上書き。
- 未割当スロットが残ったままの保存も可（一部割当として S1 に表示）。

### S8. エクスポート

**表示**
- 出力対象の選択: 元ワークブック（テンプレ）ファイルと、書き戻す集会日の範囲（通常1か月分）。
- 対象範囲に未割当スロットがある場合は警告一覧を表示（出力自体は可能。該当セルは空欄のまま）。

**操作**
- 「書き出し」: 元ワークブックの該当 E列へ名前を追記（敬称なし、2枠は `/` 区切り）し、**別名で新規保存**（原本テンプレは保持 → §4.5）。保存先はファイルダイアログで指定。
- 完了後、出力ファイルのパスを表示し、S1 の該当日ステータスを「エクスポート済み」に更新。

### S9. バックアップ・設定

**表示**
- 最終自動保存（IndexedDB ＋ data.json）の日時。
- データ保存先（data.json のパス）。

**操作**
- 「バックアップを書き出す」: 全データを JSON ファイルとして任意の場所へ保存（§3）。
- 「バックアップから復元」: JSON ファイルを選択して全データを置き換え（実行前に現在データが失われる旨の確認ダイアログ）。

## 8. セットアップ順序

1. 成員を登録（氏名・性別・状態）。
2. ロール／ロールグループ／優先度グループを定義し、成員へ付与。
3. 直近3か月の PDF から履歴をインポート（名寄せ）。
4. 割り当て開始。

## 9. v1 スコープ外（作らない）

- 公平性の俯瞰ビュー（各成員のロール別 最終担当日/通算回数一覧）。**不要**。
- PDF 直接出力（Excel からの手動 PDF 化に委ねる）。
- 補助教室（第2クラス）の別割り当て。**将来拡張の余地のみ確保**（会場属性）。
- 履歴の遡及一括修正機能。
- マルチユーザー・クラウド同期・認証・サーバ。

## 10. 実装着手時の検証事項（残リスク）

- **過去 PDF の実物**を1つ読み、テキスト抽出時の行・列の並びを検証してからパーサを確定する（「抽出可」でも列対応が崩れる場合がある）。
- **開会の祈り**の書き戻し先セルを、テンプレに追加済みの最新ファイルで確認する。
- **巡回訪問週**（奉仕の話＝非割当）・**特別キャンペーン**等の臨時項目を、レビュー画面で正しく除外/分類できること。
- **part6「話」**の回で相手役スロットを省略できること。
- ExcelJS による**書式保持の書き戻し**を実ファイルで確認。
